## XAMPP

- das Projekt im htdocs Ordner erstellen

## Projekt erstellen

- Composer und Node.js installieren
- Ein neues Laravel-Projekt in der CMD erstellen 
(zuvor ins richtige Verzeichnis wechseln: für XAMPP im htdocs Ordner)

```jsx
composer create-project laravel/laravel <Name des Projekts>
```

- Laravel-Projekt starten

```jsx
cd example-app

php artisan server
```

Funktionierender Server

- Sobald der Server gestartet ist, ist die Website unter [**localhost:8000**](http://localhost:8000) aufrufbar

## Routes

Damit lassen sich die PHP-Files im Browser visualisieren. Geschieht normal in der **routes/web.php**.

```php
<?php

use Illuminate\Support\Facades\Route;

/*
|--------------------------------------------------------------------------
| Web Routes
|--------------------------------------------------------------------------
|
| Here is where you can register web routes for your application. These
| routes are loaded by the RouteServiceProvider within a group which
| contains the "web" middleware group. Now create something great!
|
*/

Route::get('/', function () {
    return view('welcome');
});

Auth::routes();

Route::get('/home', [App\Http\Controllers\HomeController::class, 'index'])->name('home');
```

Hier ist ein Verweis auf **resources/views/welcome.blade.php**. Beim Aufruf ist die Kurzform **welcome** zu verwenden.

## Blade Templates

Um Code nicht mehrmals schreiben zu müssen, kann man bestimmte Codeteile auslagern. Das geht mit **@yield** und **@section**.

- ein **main.blade.php** in **layouts** anlegen, um das Main-Layout zu definieren für jede Seite

```php
<!doctype html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Laravel - @yield('title')</title>
</head>
<body>

<ul>
    <li><a href="/">Dashboard</a></li>
    <li><a href="/projects">Projects</a></li>
    <li><a href="/customers">Customers</a></li>
</ul>

@yield('content')

</body>
</html>
```

- in dashboard.blade.php, products.blade.php und customers.blade.php das selbe einfügen

```php
// Dashboard
@extends('layouts/main')

@section('title', 'Dashboard')

@section('content')
    <h1>Dashboard</h1>
@endsection
```

```php
// Products
@extends('layouts/main')

@section('title', 'Dashboard')

@section('content')
    <h1>Products</h1>
@endsection
```

```php
// Customers
@extends('layouts/main')

@section('title', 'Dashboard')

@section('content')
    <h1>Dashboard</h1>
@endsection
```

- testen, ob alles funktioniert

```php
php artisan serve
```

## Daten an die View übermitteln

- In der routes/web.php der Route hardcoded Daten mitgeben

```php
Route::get('/', function () {
    $tasks = [
        "INSY lernen",
        "SEW HÜ erledigen"
    ];

    return view('dashboard', ['tasks' => $tasks]);
});
```

- in der dashboard.blade.php ist das Array automatisch verfügbar und kann sofort ausgegeben werden

```php
@extends('layouts/main')

@section('title', 'Dashboard')

@section('content')
    <h1>Dashboard</h1>
    <?php var_dump($tasks); ?>
@endsection
```

## Controller

- einen **PagesController** im Verzeichnis **app/http/controllers** erstellen

```php
php artisan make:controller PagesController
```

- in den PagesController wecheln; dort steht schon folgender Code drinnen

```php
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class PagesController extends Controller
{
    //
}
```

- die Dashboard Methode hinzufügen

```php
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class PagesController extends Controller
{
    public function dashboard()
    {
        $tasks = [
            "lernen",
            "zusammenräumen"
        ];

        return view("dashboard", ["tasks" => $tasks]);
    }
}
```

- in der routes/web.php wird jetzt nur mehr ein Verweis auf den PagesController hineingeschrieben

```php
Route::get('/', [\App\Http\Controllers\PagesController::class, 'dashboard']);
```

- gleiches auch für Products und Customers machen

```php
// PagesController
public function products()
{
    return view("products");
}

public function customers()
{
    return view("customers");
}
```

```php
// routes/web.php
Route::get('/', [\App\Http\Controllers\PagesController::class, 'products']);

Route::get('/', [\App\Http\Controllers\PagesController::class, 'customers']);
```

## Datenbanken und Migrations

- Eine Migration schaut ca. so aus: 
(Code von database/migrations/…create_users_table.php)

```php
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    /**
     * Run the migrations.
     *
     * @return void
     */
    public function up()
    {
        Schema::create('users', function (Blueprint $table) {
            $table->id();
            $table->string('name');
            $table->string('email')->unique();
            $table->timestamp('email_verified_at')->nullable();
            $table->string('password');
            $table->rememberToken();
            $table->timestamps();
        });
    }

    /**
     * Reverse the migrations.
     *
     * @return void
     */
    public function down()
    {
        Schema::dropIfExists('users');
    }
};
```

- Führt man den Befehl ***php artisan migrate*** aus, wird eine Tabelle mit den Namen users und den definierten Spalten erstellt. (sowie oben im File definiert)

```php
php artisan make:migration create_customers_table
```

- Jetzt gibt es eine Datei im database/migrations Verzeichnis mit den Namen …create_customers_table.php
- die up-Methode kann man beliebig ändern; hier ein Beispiel:

```php
public function up()
    {
        Schema::create('customers', function (Blueprint $table) {
            $table->id();
            $table->string("first_name");
            $table->string("last_name");
            $table->string("address");
            $table->string("zip_code");
            $table->string("city");
            $table->string("phone");
            $table->timestamp("created_at")->nullable();
            $table->timestamp("updated_at")->nullable();
        });
    }
```

## Models erstellen

- Model erstellen

```php
php artisan make:model Customer
```

- der Befehl erstellt ein Model im Verzeichnis app/Models mit dem Namen Customer.php
- Code in Customer.php

```php
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;

class Customer extends Model
{
    use HasFactory;
}
```

- Testen, ob das Model funktioniert

```php
php artisan tinker

App\Customer::all();
```
