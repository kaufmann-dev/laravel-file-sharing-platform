# Livewire

## Livewire installieren

- PHP einfügen

```php
composer require livewire/livewire
```

- JavaScript einbinden
(bei jeder Seite, wo es gebraucht wird)

```html
...
    @livewireStyles
</head>
<body>
    ...
 
    @livewireScripts
</body>
</html>
```

## Components erstellen

- eine neue Component erstellen

```php
php artisan make:livewire <Name der Component>
```

- dieser Code ist schon vorhanden in **app/http/Livewire/Counter.php**

```php
<?php

namespace App\Http\Livewire;

use Livewire\Component;

class Counter extends Component
{
    public function render()
    {
        return view('livewire.counter');
    }
}
```

- dieser Code ist schon vorhanden in **resources/views/livewire/counter.blade.php**

```html
<div>
    {{-- If your happiness depends on money, you will never be happy with yourself. --}}
</div>
```

- den Code in **counter.blade.php** ein wenig abändern, sodass was auf der Seite steht und zu **routes/web.php** und **PagesController** hinzufügen

### Component einbinden

- JavaScript einbinden in counter.blade.php und eine Livewire-Methode hinzufügen
- Im Terminal livewire einbinden

```php
composer require livewire/livewire
```

- eine Component erstellen

```php
php artisan make:livewire counter
```

- damit werden zwei Files erstellt:
app/Http/Livewire/Counter.php
resources/views/livewire/counter.blade.php
- Zur Probe, Code in die beiden Files einfügen

```php
// app/Http/Livewire/Counter.php
namespace App\Http\Livewire;
 
use Livewire\Component;
 
class Counter extends Component
{
    public function render()
    {
        return view('livewire.counter');
    }
}
```

```html
// resources/views/livewire/counter.blade.php
<div>
    <h1>Hello World!</h1>
</div>
```

- im welcome.blade.php die Component hinzufügen

```html
// welcome.blade.php
<!DOCTYPE html>
<html>
<head>
    <title>Laravel Livewire CRUD - ItSolutionStuff.com</title>
    <link href="https://stackpath.bootstrapcdn.com/bootstrap/4.5.0/css/bootstrap.min.css" rel="stylesheet">
    @livewireStyles
</head>
<body>
    <livewire:counter />

    @livewireScripts
</body>
</html>
```

- Im Browser testen

```php
php artisan serve
```

![Untitled](Livewire%20d62c5a7f1f9949a381fc3ca4e463111c/Untitled.png)

- wenn alles funktioniert, dann kann man den richtigen Code für den Counter implementieren

```php
// app/Http/Livewire/Counter.php
<?php

namespace App\Http\Livewire;

use Livewire\Component;

class Counter extends Component
{
    public $count = 0;

    public function increment()
    {
        $this->count++;
    }

    public function render()
    {
        return view('livewire.counter');
    }
}
```

```html
// resources/views/livewire/counter.blade.php
<div style="text-align: center">
    <button wire:click="increment">+</button>
    <h1>{{ $count }}</h1>
</div>
```

- wieder im Browser testen

```php
php artisan serve
```

## Template Directives

```html
<button wire:click="<Name des Directives>"...</button>
```

| Directive | Beschreibung |
| --- | --- |
| wire:key=”foo” | Nützlich für Hinzufügen/Entfernen von Elementen und für Listen |
| wire:click=”foo” | Clickfunction, welche die Function “foo” in der Component auslöst |
| wire:click.prefetch="foo” | Wartet auf ein "mouseenter"-Ereignis und "prefetched" das Ergebnis der "foo"-Methode in der Komponente. Wenn sie dann angeklickt wird, wird das "vorgeholte" Ergebnis ausgetauscht (ohne zusätzliche Anfrage), wenn sie nicht angeklickt wird, wird das zwischengespeicherte Ergebnis verworfen. |
| wire:keydown.enter="foo” | Wartet auf ein Keydown-Ereignis auf der Eingabetaste, das die Methode "foo" in der Komponente auslöst. |
| wire:foo="bar” | Hört auf ein Browser-Ereignis namens "foo”. |
| wire:model="foo” | Unter der Annahme, dass $foo eine public property der Component ist, wird jedes Mal, wenn ein Eingabeelement mit dieser Richtlinie aktualisiert wird, die Eigenschaft mit ihrem Wert synchronisiert. |
| wire:model.debounce.100ms="foo” | Löst alle 100 Millisekunden die vom Element ausgegebenen Eingabeereignisse auf. |
| wire:model.lazy="foo” | Synchronisiert die Eingabe mit der entsprechenden Component-Eigenschaft im Ruhezustand. |
| wire:model.defer="foo” | Verschiebt die Synchronisierung der Eingabe mit der Livewire-Eigenschaft, bis eine "Aktion" durchgeführt wird. |
| wire:poll.500ms="foo” | Führt alle 500 Millisekunden die Methode "foo" für die Komponentenklasse aus. |
| wire:init="foo” | Führt die "foo"-Methode für die Komponente unmittelbar nach deren Darstellung auf der Seite aus. |
| wire:loading | Blendet das Element standardmäßig aus und macht es sichtbar, während die Seite lädt. |
| wire:loading.class="foo” | Fügt dem Element die Klasse foo hinzu, während Netzanfragen unterwegs sind. |
| wire:loading.class.remove="foo" | Entfernt die Klasse foo, während Netzwerkanfragen unterwegs sind. |
| wire:loading.attr="disabled” | Fügt das Attribut disabled="true" hinzu, wenn Netzwerkanfragen unterwegs sind. |
| wire:dirty | Blendet das Element standardmäßig aus und macht es sichtbar, wenn der Zustand des Elements "dirty" ist. |
| wire:dirty.class="foo” | Fügt dem Element die Klasse foo hinzu, solange es dirty ist. |
| wire:dirty.class.remove="foo” | Entfernt die Klasse foo, während das Element dirty ist. |
| wire:dirty.attr="disabled” | Fügt das Attribut disabled="true" hinzu, während das Element "dirty" ist. |
| wire:target="foo” | Verteilt die Funktionen wire:loading und wire:dirty auf eine bestimmte Aktion. |
| wire:ignore | Weist Livewire an, das Element oder seine Kinder nicht zu aktualisieren, wenn das DOM durch eine Serveranfrage aktualisiert wird. |
| wire:ignore.self | Der Modifikator "self" schränkt Aktualisierungen auf das Element selbst ein, erlaubt aber Änderungen an seinen Kindern. |

## Global Livewire JavaScript Objects

Die Methoden sind verfügbar im Frontend bei **window.Livewire**.

| Methode | Beschreibung |
| --- | --- |
| Livewire.first() | Abrufen des JS-Objekts der ersten Livewire-Komponente auf der Seite |
| Livewire.find(componentId) | Ermitteln einer Livewire-Komponente anhand ihrer ID |
| Livewire.all() | Alle Livewire-Komponenten auf einer Seite anzeigen |
| Livewire.directive(directiveName, (el, directive, component) => {}) | Registrieren einer neuen Livewire-Richtlinie (wire:custom-directive) |
| Livewire.hook(hookName, (...) => {}) | Aufruf einer Methode, wenn der JS-Lifecycle-Hook ausgelöst wird. |
| Livewire.onLoad(() => {}) | Wird ausgelöst, wenn Livewire das erste Mal auf einer Seite fertig geladen ist |
| Livewire.onError((message, statusCode) => {}) | Wird ausgelöst, wenn eine Livewire-Anfrage fehlschlägt. Sie können false aus dem Callback zurückgeben, um das Standardverhalten von Livewire zu verhindern |
| Livewire.onPageExpired((response, message) => {}) | Wenn die Seite oder Sitzung abgelaufen ist, wird der Callback anstelle von Livewires ausgeführt |
| Livewire.emit(eventName, ...params) | Aussenden eines Ereignisses an alle Livewire-Komponenten, die auf einer Seite zuhören |
| Livewire.emitTo(componentName, eventName, ...params) | Ein Ereignis an einen bestimmten Komponentennamen senden |
| Livewire.on(eventName, (...params) => {} | Warten auf ein Ereignis, das von einer Komponente ausgesendet wird |
| Livewire.start() | Booten Sie Livewire auf der Seite (wird für Sie automatisch über @livewireScripts erledigt) |
| Livewire.stop() | <div><br class="Apple-interchange-newline">Livewire von der Seite trennen</div> |
| Livewire.restart() | Livewire auf der Seite anhalten und dann starten |
| Livewire.rescan() | Erneutes Scannen des DOM nach neu hinzugefügten Livewire-Komponenten |

## Components rendern

[https://laravel-livewire.com/docs/2.x/rendering-components](https://laravel-livewire.com/docs/2.x/rendering-components)

- es gibt verschiedene Einbindungsmöglichkeiten

```html
// Beispiel 1
<div>
    <livewire:counter />
</div>
```

```php
// Beispiel 2
@livewire('counter')
```

- sind die Components im app/Http/Livewire/Nav Ordner wären, muss man die Component so einbinden:

```html
<livewire:nav.show-posts />
```

### Parameter übergeben

```html
// Beispiel 1
<livewire:counter :post="$post">
```

```php
// Beispiel 2
@livewire('counter', ['post' => $post])
```

- die Parameter bekommt man von Components

```php
// Beispiel 1
class Counter extends Component
{
    public $post;
 
    ...
}
```

### Full-Page Components

- Man kann eine Component über eine Route direkt zum Art Controller machen

```php
Route::get('/post', Counter::class);
```

- Standardmäßig rendert Livewire die Counter-Komponente in den {{ $slot }} einer Blade-Layout-Komponente unter: resources/views/layouts/app.blade.php
- Man kann die Component dann so verwenden:

```html
<head>
    @livewireStyles
</head>
<body>
    {{ $slot }}
 
    @livewireScripts
</body>
```
